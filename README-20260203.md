- Windows 10/11 (64-bit)

### Installation Steps

#### 1. Check Node.js
```cmd
node --version
```
If not installed, please download from [https://nodejs.org](https://nodejs.org).

#### 2. Install OpenClaw Daemon
Copy all files from `OpenClaw-Daemon/` folder to your OpenClaw root directory.

#### 3. Start OpenClaw Daemon
First start the OpenClaw daemon for reliable service:
```cmd
# In your OpenClaw directory
Start OpenClaw Daemon.bat
```

#### 4. Start Crabwalk
Now double-click `Start Crabwalk.bat` to launch the session monitor!

#### 5. Access Interface
Open your browser and visit: http://localhost:3000

## ğŸ”§ Configuration

### Default Configuration
- **Gateway URL**: `ws://127.0.0.1:18789`
- **Auth Token**: Pre-configured
- **Default Session**: `agent:main:main`
- **Service Port**: `3000`

### Custom Configuration
If you need to modify settings, edit the environment variables in `Start Crabwalk.bat`:

```batch
set CLAWDBOT_URL=ws://127.0.0.1:18789
set CLAWDBOT_API_TOKEN=your_token_here
set CLAWDBOT_DEFAULT_SESSION=agent:main:main
```

## ğŸ› Troubleshooting

### Issue 1: Port 3000 is occupied
```cmd
# Check which process is using it
netstat -ano | findstr :3000

# Force kill the process
taskkill /PID process_id /F
```

### Issue 2: Cannot connect to OpenClaw
1. Make sure OpenClaw daemon is running first: `Start OpenClaw Daemon.bat`
2. Check if OpenClaw is running: `netstat -ano | findstr :18789`
3. Generate a new authentication token: `openclaw dashboard --no-open`
4. Update the token in the `Start Crabwalk.bat` script

### Issue 3: Node.js not installed
Download and install Node.js from [nodejs.org](https://nodejs.org).

### Issue 4: No session data visible
1. Ensure OpenClaw has active sessions: `openclaw sessions list`
2. Check if the authentication token is correct
3. Restart the Crabwalk service

## ğŸ“ Technical Details

### Environment Variables
- `CLAWDBOT_URL`: OpenClaw gateway WebSocket address
- `CLAWDBOT_API_TOKEN`: Authentication token
- `CLAWDBOT_DEFAULT_SESSION`: Default monitored session

### Data Storage
- Session data: `data/sessions.json`
- Action logs: `data/actions.jsonl`
- Execution events: `data/exec-events.jsonl`
- Status info: `data/state.json`
